## Codebase Patterns
- Active Ralph run files live under `scripts/ralph/`; `ralph.sh` reads `scripts/ralph/prd.json` and `scripts/ralph/progress.txt`
- The repo root `prd.json` can mirror the active iteration for humans, but the autonomous loop follows `scripts/ralph/prd.json`
- Existing LAN discovery already exists on all three clients: macOS `BonjourBrowser`, Windows `MdnsDiscoveryService`, Android `MdnsDiscoveryService`
- Current discovered-server models use host and port for display identity, but DHCP-safe persistence should use a stable server-side `server_id`
- Manual Host URL entry remains the fallback path; new discovery work should not delete or block a valid manual configuration
- Multi-server discovery should remain passive: do not add modal choosers or startup popups; guide the user to Config for explicit selection
- This iteration only guarantees discovery restore on app restart or LAN mode re-entry, not live hot switching while already connected
