## Codebase Patterns
- Active Ralph run files live under `scripts/ralph/`; `ralph.sh` reads `scripts/ralph/prd.json` and `scripts/ralph/progress.txt`
- The repo root `prd.json` can mirror the active iteration for humans, but the autonomous loop follows `scripts/ralph/prd.json`
- Targeted LAN delivery already uses `server`-stored clips plus SSE notify plus client `GET /clips/:id/raw`; do not introduce client-side inbound APIs unless a later PRD explicitly changes the architecture
- Server delivery-state lifecycle starts targeted clips in `targeted_pending`; only untargeted clips use `ready` immediately, so new server/client work should not assume every sendable clip is `ready`
- Timeout-driven targeted fallback is keyed off persisted `targeted_pending_at`; set it only when a clip actually enters `targeted_pending` (single upload or chunked complete), not when a chunked upload is first initialized
- Server targeted SSE events are emitted from `server/handlers/clips.go` and `server/handlers/uploads.go`, and replayed on connect from `server/handlers/devices.go`
- SSE reconnect replay should query `targeted_pending` clips from SQLite in oldest-first order so reconnects preserve delivery attempts without turning replay into a new delivery state
- `/api/v1/devices` receiver presence is computed from the live `server/sse.Broker`, not persisted in SQLite; fresh active SSE subscriptions are `online`, stale active subscriptions are `degraded`, and no subscriber is `offline`
- Targeted `GET /api/v1/clips/:id/raw` now requires `?device_id=<target>` and is the authoritative delivery acknowledgement path; only that addressed-device consume should transition a clip to `targeted_delivered`
- Consumed clip rows should remain in SQLite until cleanup runs so duplicate raw consumes keep returning `410 Gone` and targeted delivery state stays inspectable after the first successful read
- macOS targeted auto-receive lives in `macos/CopyEverywhere/Sources/CopyEverywhere/ConfigStore.swift`
- Windows targeted auto-receive lives in `windows/CopyEverywhere/MainWindow.xaml.cs`
- Android targeted auto-receive lives in `android/app/src/main/java/com/copyeverywhere/app/service/CopyEverywhereService.kt`
