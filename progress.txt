## Codebase Patterns
- Use `modernc.org/sqlite` (pure Go, no CGO) for SQLite
- Database package lives in `server/db/` with `Open(storagePath)` constructor
- Clip model uses `*string` for nullable fields (e.g., Filename)
- ID generation: 6-char alphanumeric via crypto/rand
- Tests use `t.TempDir()` for isolated DB instances
- SQLite connection uses `SetMaxOpenConns(1)` for write safety
- Handlers live in `server/handlers/` with struct-based dependency injection (DB, config)
- Gin route order matters: `/clips/latest` must be registered before `/clips/:id` to avoid param conflict
- Use `c.File()` for streaming file content in Gin (handles range requests automatically)
- Clip API returns `clipResponse` (subset of Clip) to avoid leaking internal fields like StoragePath
- Chunked uploads store parts in `STORAGE_PATH/uploads/<id>/part_N`, merged on complete to `STORAGE_PATH/<id>/<filename>`
- `UpdateClip(id, status, sizeBytes, storagePath)` updates clip record after chunked upload completes
- Cleanup package in `server/cleanup/` runs background goroutine with `Start(db, storagePath, interval)`
- `GetExpiredClips()` returns all clips where `expires_at < now()` (both ready and uploading)
- `GetStorageStats()` returns `{ClipCount, StorageUsedBytes}` via SQL COUNT/SUM
- macOS app uses Swift Package Manager (not .xcodeproj) in `macos/CopyEverywhere/`
- Build with `swift build` from `macos/CopyEverywhere/` directory
- MenuBarExtra with `.window` style creates popover panel from menu bar icon
- `NSApp.setActivationPolicy(.accessory)` hides dock icon programmatically
- Keychain access via Security framework: `SecItemAdd`/`SecItemCopyMatching`/`SecItemDelete`
- ConfigStore is `@MainActor` ObservableObject shared via `.environmentObject()`
- Use `UserNotifications` framework (not deprecated `NSUserNotification`) for macOS notifications
- Receive flow: first GET /api/v1/clips/latest for metadata, then GET /api/v1/clips/:id/raw for content
- File upload uses `URLSession.upload(for:fromFile:)` with `URLSessionTaskDelegate` for progress tracking
- `UploadProgressDelegate` bridges delegate callbacks to `AsyncStream<Double>` for async/await consumption
- 50MB threshold separates small file (single POST) from chunked upload flow
- Chunked upload flow: init → upload parts (1-indexed) → complete; pause/resume via server status query
- `FileHandle` for reading file chunks without loading entire file into memory
- File download uses `URLSession.download(for:)` with `URLSessionDownloadDelegate` for streaming + progress
- `DownloadProgressDelegate` yields `(totalBytesWritten, totalBytesExpectedToWrite)` via `AsyncStream`
- `NSSavePanel` for choosing download save location; `NSWorkspace.shared.activateFileViewerSelecting` to reveal in Finder
- Clip metadata API (`GET /clips/:id`) returns `clipResponse` without `status` field; `/raw` returns 403 for failed uploads
- HistoryStore uses UserDefaults with JSON encoding (Codable) for local history persistence
- ConfigStore receives `historyStore` reference via MenuBarView `.onAppear` (not init param) to work with @EnvironmentObject
- Windows project in `windows/CopyEverywhere/` uses .NET 8 WPF with `Hardcodet.NotifyIcon.Wpf` for system tray
- Windows config stored in Windows Credential Manager via `CredentialManagement` NuGet package
- Windows WPF PasswordBox doesn't support data binding — use `PasswordChanged` event handler to sync
- `ShutdownMode="OnExplicitShutdown"` in App.xaml prevents app from closing when main window hides to tray
- Windows toast notifications via `Microsoft.Toolkit.Uwp.Notifications` NuGet (works with .NET 8 WPF)
- ApiClient uses `MultipartFormDataContent` for clip uploads matching server's multipart/form-data expectation
- Clipboard text send uses "clipboard.txt" filename with "text/plain" content type
- Receive flow: GET /clips/latest for metadata → GET /clips/:id/raw for content (same as macOS)
- `ProgressStreamContent` (extends HttpContent) for tracking upload progress in .NET — no built-in alternative
- WPF file dialogs (`OpenFileDialog`, `SaveFileDialog`) are in `Microsoft.Win32` namespace
- `Process.Start("explorer.exe", "/select,\"path\"")` reveals file in Explorer
- Chunked upload 409 from PUT parts means chunk already uploaded — treat as success during resume
