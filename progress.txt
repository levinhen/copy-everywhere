# Embedded Server — Progress

## Codebase Patterns
- ServerProcess from CopyEverywhereServer is already `@MainActor` + `ObservableObject` — copies directly into the client app without adaptation.
- ServerProcess depends on ServerConfig for its `environment` property — need at least a stub for compilation.
- AppDelegate owns top-level singletons (configStore, serverConfig, serverProcess) and injects them via `@EnvironmentObject`.
- New ConfigData fields must be `Optional` with defaults in `load()` for backward compat with older config files.

## 2026-04-13 - US-071

- Auto-start on launch: `applicationDidFinishLaunching` checks `serverEnabled && autoStartServer`
- Injected `serverConfig` and `serverProcess` as `@EnvironmentObject` into the view hierarchy for use by US-072
- Files changed:
  - macos/CopyEverywhere/Sources/CopyEverywhere/ServerConfig.swift (added serverEnabled, autoStartServer + persistence)
  - macos/CopyEverywhere/Sources/CopyEverywhere/CopyEverywhereApp.swift (added serverConfig, toggle logic, auto-start, env objects)
- **Learnings for future iterations:**
  - AppDelegate owns all three singletons (configStore, serverConfig, serverProcess) — toggle logic lives there because it needs all three
  - `autoConnectToLocalServer()` calls `configStore.save()` which triggers device registration + SSE restart — no extra wiring needed
  - New ConfigData fields must be optional (`Bool?`) with defaults in `load()` for backward compatibility with existing config files
