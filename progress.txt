## Codebase Patterns
- Windows: WPF PasswordBox doesn't support two-way binding â€” sync via PasswordChanged event
- Windows: Use Application.Current.Dispatcher.Invoke() to update ObservableCollection from background threads (OutputDataReceived fires on a thread-pool thread)
- Windows: Process.Kill(entireProcessTree: true) available on .NET 5+ â€” use it to ensure child processes are also killed
- Windows: AppContext.BaseDirectory gives the directory containing the EXE (equivalent to Swift's Bundle.main.bundlePath)
- Windows: Registry reads/writes via Microsoft.Win32.Registry.CurrentUser.OpenSubKey / CreateSubKey / DeleteValue â€” no extra NuGet needed
- Windows: FolderBrowserDialog is in System.Windows.Forms â€” add <UseWindowsForms>true</UseWindowsForms> to csproj or use a WPF-native CommonOpenFileDialog from Microsoft.WindowsAPICodePack if available
- Windows: Process.StartInfo.Environment is a Dictionary<string,string> pre-populated with the current process environment; override specific keys before starting
- Windows: Atomic file write pattern: write to <path>.tmp then File.Replace(<path>.tmp, <path>, null)
- Windows: GetFolderPath(Environment.SpecialFolder.ApplicationData) returns %APPDATA% (roaming); LocalApplicationData returns %LOCALAPPDATA%
- Server binary path convention: copyeverywhere-server.exe placed as a sibling next to CopyEverywhere.exe (same directory); compiled separately with go build -o copyeverywhere-server.exe . from server/
