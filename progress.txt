## Codebase Patterns
- Gradle version catalog at `android/gradle/libs.versions.toml` — always use `libs.` references
- `./gradlew assembleDebug` from `android/` is the build check command
- Android package is `com.copyeverywhere.app`
- Adaptive icons via vector drawables in `mipmap-anydpi-v26/` — no raster PNGs needed
- `dependencyResolutionManagement` in settings.gradle.kts (not `dependencyResolution` — that's Gradle 9+ only)
- DataStore Preferences for non-secret config, EncryptedSharedPreferences (Keystore-backed) for access token
- OkHttp + Gson for API calls; ApiClient class in `data/` package
- Navigation Compose for screen navigation (`NavHost` + `composable()` routes)
- `ExposedDropdownMenuBox` with `MenuAnchorType.PrimaryNotEditable` for read-only dropdowns in Material 3
- Server `/api/v1/devices/register` currently only accepts macos/windows/linux — Android client sends "android", server needs update
- mDNS discovery via `NsdManager` in `MdnsDiscoveryService` — use `DisposableEffect` for start/stop lifecycle in Compose
- `NsdServiceInfo.attributes` returns TXT record fields as `Map<String, ByteArray>` — decode with `String(bytes)`
- Chunked uploads use separate OkHttpClient with 120s timeout; `ChunkedUploadState` holds progress as `MutableStateFlow<Double>`
- `currentCoroutineContext().ensureActive()` for cooperative cancellation in coroutine loops
- SSE client: OkHttp with `readTimeout(0)` for infinite timeout, `BufferedReader` line-by-line parsing
- Foreground service: `FOREGROUND_SERVICE` + `FOREGROUND_SERVICE_DATA_SYNC` permissions, `foregroundServiceType="dataSync"` in manifest
- Notification channels: `CHANNEL_SERVICE` (low importance, ongoing) and `CHANNEL_TRANSFERS` (high importance, auto-cancel)
- Android 13+ requires `POST_NOTIFICATIONS` runtime permission — request via `ActivityResultContracts.RequestPermission()`
- `CopyEverywhereService.start(context)` / `.stop(context)` static helpers for service lifecycle
- Share sheet: `ShareReceiverActivity` handles ACTION_SEND (single) and ACTION_SEND_MULTIPLE; routes to ApiClient single or chunked upload
- `ActivityResultContracts.OpenDocument()` for system file picker; pass `arrayOf("*/*")` for all types
- `Icons.AutoMirrored.Filled.Send` (not `Icons.Default.Send` — deprecated)
- File save to Downloads: use `MediaStore.Downloads` (no storage permissions needed on API 29+)
- Android 10+ clipboard restriction: use transparent `ClipboardTrampolineActivity` to read clipboard from notification actions (background services can't read clipboard)
- `buildServiceNotificationStatic()` on `CopyEverywhereService.Companion` for updating the service notification from outside the service
- `CopyEverywhereService.instance` static reference for ViewModel→Service communication (e.g., `switchMode()`)
- `TransferMode` enum in `ConfigStore` — check `transferMode.value` before starting LAN-only operations (queue polling, SSE)
- Bluetooth RFCOMM: `BluetoothProtocol.SERVICE_UUID` = `CE000001-1000-1000-8000-00805F9B34FB` — must match all platforms
- `BluetoothSession` wraps `BluetoothSocket` — handshake + receive loop on `Dispatchers.IO`, progress via `Flow<Double>`
- `BluetoothService` manages RFCOMM server (`listenUsingRfcommWithServiceRecord`) + client (`createRfcommSocketToServiceRecord`)
- `@SuppressLint("MissingPermission")` on BluetoothService — runtime permission checks done in UI layer
- `CopyEverywhereService.bluetoothService` is the live instance; `startBluetoothServer()`/`stopBluetoothServer()` in `switchMode()`
- Bluetooth permissions: `BLUETOOTH`+`BLUETOOTH_ADMIN` (maxSdkVersion 30), `BLUETOOTH_CONNECT`+`BLUETOOTH_SCAN` (API 31+)
- Paired Bluetooth devices persisted in DataStore as JSON via Gson (`PairedBluetoothDevice` data class in ConfigStore)
- `BluetoothAdapter.startDiscovery()` + `BroadcastReceiver(ACTION_FOUND)` for device scanning — register with `RECEIVER_EXPORTED` on API 33+
- `Icons.AutoMirrored.Filled.BluetoothSearching` (not `Icons.Default.BluetoothSearching` — deprecated)
- `BluetoothService.listener` shared between ConfigViewModel and Service — restore Service as listener in ViewModel's `onCleared()` (not null)
- `BluetoothService.autoReconnect()` checks `isSessionReady` before each attempt — inbound RFCOMM server accept can satisfy the reconnect
- All BT send entry points: check transferMode → `CopyEverywhereService.instance?.bluetoothService?.activeSession` → verify `isHandshakeComplete` → send
- `BluetoothSession.sendText()`/`sendFile()` return cold `Flow<Double>` — must `collect` to execute the send
