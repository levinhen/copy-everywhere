# Embedded Server — Progress

## Codebase Patterns
- ServerProcess from CopyEverywhereServer is already `@MainActor` + `ObservableObject` — copies directly into the client app without adaptation.
- ServerProcess depends on ServerConfig for its `environment` property — need at least a stub for compilation.
- AppDelegate owns top-level singletons (configStore, serverConfig, serverProcess) and injects them via `@EnvironmentObject`.
- New ConfigData fields must be `Optional` with defaults in `load()` for backward compat with older config files.
- When restarting the Go subprocess, poll `isRunning` instead of fixed delay — mDNS needs the old process fully exited before the new one starts.
