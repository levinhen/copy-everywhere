## Codebase Patterns
- Gradle version catalog at `android/gradle/libs.versions.toml` ‚Äî always use `libs.` references
- `./gradlew assembleDebug` from `android/` is the build check command
- Android package is `com.copyeverywhere.app`
- Adaptive icons via vector drawables in `mipmap-anydpi-v26/` ‚Äî no raster PNGs needed
- `dependencyResolutionManagement` in settings.gradle.kts (not `dependencyResolution` ‚Äî that's Gradle 9+ only)
- DataStore Preferences for non-secret config, EncryptedSharedPreferences (Keystore-backed) for access token
- OkHttp + Gson for API calls; ApiClient class in `data/` package
- Navigation Compose for screen navigation (`NavHost` + `composable()` routes)
- `ExposedDropdownMenuBox` with `MenuAnchorType.PrimaryNotEditable` for read-only dropdowns in Material 3
- Server `/api/v1/devices/register` currently only accepts macos/windows/linux ‚Äî Android client sends "android", server needs update
- mDNS discovery via `NsdManager` in `MdnsDiscoveryService` ‚Äî use `DisposableEffect` for start/stop lifecycle in Compose
- `NsdServiceInfo.attributes` returns TXT record fields as `Map<String, ByteArray>` ‚Äî decode with `String(bytes)`
- Chunked uploads use separate OkHttpClient with 120s timeout; `ChunkedUploadState` holds progress as `MutableStateFlow<Double>`
- `currentCoroutineContext().ensureActive()` for cooperative cancellation in coroutine loops

- Files modified:
  - `android/app/src/main/java/com/copyeverywhere/app/data/ApiClient.kt`
- **Learnings for future iterations:**
  - OkHttp `MultipartBody.Builder` handles boundary generation automatically ‚Äî no hand-rolling needed like macOS Swift
  - Use okio `source()` on InputStream + `sink.writeAll()` for streaming file uploads without loading into memory
  - Custom `RequestBody` subclass with `writeTo(BufferedSink)` is the pattern for streaming uploads from content URIs
  - `ContentResolver.query()` with `OpenableColumns.DISPLAY_NAME` and `OpenableColumns.SIZE` to get file metadata from URIs
  - `MimeTypeMap.getSingleton().getMimeTypeFromExtension()` for MIME type guessing from filename extension
  - Image clips use type="image", regular files use type="file" ‚Äî matches server expectations
  - `getFileName()`, `getFileSize()`, `guessMimeType()` placed in companion object for reuse in chunked upload (US-055)

## 2026-04-12 - US-055
- Implemented chunked upload for files >= 50 MB in ApiClient
- `initChunkedUpload()`: POST /uploads/init with filename, size_bytes, chunk_size, device IDs
- `uploadChunks()`: streams each chunk via PUT /uploads/:id/parts/:n with 16 KB read buffer
- `completeChunkedUpload()`: POST /uploads/:id/complete returns clip ID
- `getUploadStatus()` + `resumeChunkedUpload()`: query server for completed parts to resume
- `ChunkedUploadState` class holds upload state, progress (Flow<Double>), speed (Flow<Double> MB/s)
- 409 response treated as success (chunk already uploaded)
- Pause = cancel coroutine Job; resume = query status + re-call uploadChunks
- Separate `uploadClient` with 120s read/write timeout for chunk transfers
- Files modified:
  - `android/app/src/main/java/com/copyeverywhere/app/data/ApiClient.kt`
- **Learnings for future iterations:**
  - Chunk uploads need longer timeouts than regular API calls ‚Äî use separate OkHttpClient
  - `InputStream.skip()` may not skip the full amount in one call ‚Äî loop until offset reached
  - `currentCoroutineContext().ensureActive()` is the pattern for cooperative cancellation in loops
  - `ChunkedUploadState` uses MutableStateFlow for progress/speed so Compose can collect them
  - Default chunk size 5 MB balances between too many HTTP requests and memory usage
