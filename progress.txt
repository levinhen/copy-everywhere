## Codebase Patterns
- Use `modernc.org/sqlite` (pure Go, no CGO) for SQLite
- Database package lives in `server/db/` with `Open(storagePath)` constructor
- Clip model uses `*string` for nullable fields (e.g., Filename)
- ID generation: 6-char alphanumeric via crypto/rand
- Tests use `t.TempDir()` for isolated DB instances
- SQLite connection uses `SetMaxOpenConns(1)` for write safety
- Handlers live in `server/handlers/` with struct-based dependency injection (DB, config)
- Gin route order matters: `/clips/latest` must be registered before `/clips/:id` to avoid param conflict
- Use `c.File()` for streaming file content in Gin (handles range requests automatically)
- Clip API returns `clipResponse` (subset of Clip) to avoid leaking internal fields like StoragePath
- Chunked uploads store parts in `STORAGE_PATH/uploads/<id>/part_N`, merged on complete to `STORAGE_PATH/<id>/<filename>`
- `UpdateClip(id, status, sizeBytes, storagePath)` updates clip record after chunked upload completes
- Cleanup package in `server/cleanup/` runs background goroutine with `Start(db, storagePath, interval)`
- `GetExpiredClips()` returns all clips where `expires_at < now()` (both ready and uploading)
- `GetStorageStats()` returns `{ClipCount, StorageUsedBytes}` via SQL COUNT/SUM
- macOS app uses Swift Package Manager (not .xcodeproj) in `macos/CopyEverywhere/`
- Build with `swift build` from `macos/CopyEverywhere/` directory
- MenuBarExtra with `.window` style creates popover panel from menu bar icon
- `NSApp.setActivationPolicy(.accessory)` hides dock icon programmatically
- Keychain access via Security framework: `SecItemAdd`/`SecItemCopyMatching`/`SecItemDelete`
- ConfigStore is `@MainActor` ObservableObject shared via `.environmentObject()`
- Use `UserNotifications` framework (not deprecated `NSUserNotification`) for macOS notifications
- Receive flow: first GET /api/v1/clips/latest for metadata, then GET /api/v1/clips/:id/raw for content
- File upload uses `URLSession.upload(for:fromFile:)` with `URLSessionTaskDelegate` for progress tracking
- `UploadProgressDelegate` bridges delegate callbacks to `AsyncStream<Double>` for async/await consumption
- 50MB threshold separates small file (single POST) from chunked upload flow
- Chunked upload flow: init → upload parts (1-indexed) → complete; pause/resume via server status query
- `FileHandle` for reading file chunks without loading entire file into memory
- File download uses `URLSession.download(for:)` with `URLSessionDownloadDelegate` for streaming + progress
- `DownloadProgressDelegate` yields `(totalBytesWritten, totalBytesExpectedToWrite)` via `AsyncStream`
- `NSSavePanel` for choosing download save location; `NSWorkspace.shared.activateFileViewerSelecting` to reveal in Finder
- Clip metadata API (`GET /clips/:id`) returns `clipResponse` without `status` field; `/raw` returns 403 for failed uploads
- HistoryStore uses UserDefaults with JSON encoding (Codable) for local history persistence
- ConfigStore receives `historyStore` reference via MenuBarView `.onAppear` (not init param) to work with @EnvironmentObject
- Windows project in `windows/CopyEverywhere/` uses .NET 8 WPF with `Hardcodet.NotifyIcon.Wpf` for system tray
- Windows config stored in Windows Credential Manager via `CredentialManagement` NuGet package
- Windows WPF PasswordBox doesn't support data binding — use `PasswordChanged` event handler to sync
- `ShutdownMode="OnExplicitShutdown"` in App.xaml prevents app from closing when main window hides to tray
- Windows toast notifications via `Microsoft.Toolkit.Uwp.Notifications` NuGet (works with .NET 8 WPF)
- ApiClient uses `MultipartFormDataContent` for clip uploads matching server's multipart/form-data expectation
- Clipboard text send uses "clipboard.txt" filename with "text/plain" content type
- Receive flow: GET /clips/latest for metadata → GET /clips/:id/raw for content (same as macOS)
- `ProgressStreamContent` (extends HttpContent) for tracking upload progress in .NET — no built-in alternative
- WPF file dialogs (`OpenFileDialog`, `SaveFileDialog`) are in `Microsoft.Win32` namespace
- `Process.Start("explorer.exe", "/select,\"path\"")` reveals file in Explorer
- Chunked upload 409 from PUT parts means chunk already uploaded — treat as success during resume
- Windows HistoryStore uses JSON file in AppData/Local/CopyEverywhere/history.json (mirrors macOS UserDefaults approach)
- WPF history list built programmatically with DockPanel rows, Border.MouseLeftButtonUp for click-to-copy, Button.Tag for passing ClipId
- Windows device config (device_id, device_name) persisted as JSON in %LOCALAPPDATA%\CopyEverywhere\config.json — not in Credential Manager (not a secret)
- Windows device registration on save: best-effort POST /devices/register in SaveButton_Click, exceptions swallowed
- GET /api/v1/clips/:id/raw is now an atomic consume: first caller wins, second gets 410 Gone; async goroutine cleans up files/DB after response
- GET /api/v1/clips?device_id=XXX returns the queue view; GET /api/v1/clips/latest is deprecated (sends Deprecation header)
- Device IDs are 8-char alphanumeric (vs 6-char for clip IDs); both use `generateRandomID(length)` in `db/db.go`
- Device registration is idempotent on (name, platform) — re-registering returns the existing ID and bumps `last_seen_at`
- DeviceHandler is a separate struct from ClipHandler/UploadHandler, wired in main.go
- Device routes: POST /api/v1/devices/register, GET /api/v1/devices
- Clip columns: target_device_id (*string), sender_device_id (*string), consumed_at (*time.Time) — all nullable
- SQLite migration uses `ALTER TABLE ADD COLUMN` with errors ignored for idempotence (no IF NOT EXISTS syntax)
- POST /clips takes device IDs as multipart form fields; POST /uploads/init takes them as JSON body fields
- SSE broker lives in `server/sse/` with `NewBroker()`, `Subscribe(deviceID)`, `Unsubscribe(deviceID, ch)`, `Notify(deviceID, event)`
- SSE subscriber channels have buffer size 16; overflow events are dropped (non-blocking send)
- SSE stream handler sends `: connected\n\n` on open, then loops on event channel + 25s heartbeat ticker
- SSE tests use `httptest.NewServer` (not `httptest.NewRecorder`) because SSE requires streaming response
- Broker, ClipHandler, and UploadHandler all share the same `*sse.Broker` instance wired in main.go
- MenuBarExtra replaced with manual NSStatusItem + NSPopover in AppDelegate for drag-and-drop support
- Queue listing via GET /api/v1/clips?device_id=XXX; each item has {id, type, filename, size_bytes, created_at, expires_at}
- Queue receive is atomic via GET /clips/:id/raw: first caller gets body (200), second gets 410 Gone; clip is auto-deleted after consume
- QueueItem model and fetchQueue()/receiveQueueItem() live in ConfigStore (no separate ApiClient class)
- MainPanelView uses Timer.scheduledTimer for 5s queue refresh while panel is open; invalidated on disappear
- HistoryStore.swift deleted in US-027 — no local history; the server queue is the single source of truth
- StatusItemDropView is an NSView overlay on NSStatusBarButton — hitTest returns nil so clicks pass through, drag destination still works (AppKit resolves via frame, not hitTest)
- AppDelegate must be `@MainActor` to access `@MainActor`-isolated ConfigStore/HistoryStore properties synchronously
- NSPopover with `.applicationDefined` behavior + global event monitor avoids the double-toggle issue with `.transient`
- `sendText(_:)` is a fire-and-forget text send (returns tuple) that doesn't touch `sendStatus` — separate from `sendClipboardText()` which updates UI state
- File drops reuse existing `sendFile(url:)` which handles both small (<50MB) and chunked (>=50MB) paths
- `AppDelegate.sendNotification(body:)` is static; checks `Bundle.main.bundleIdentifier != nil` to avoid crashes in unbundled SPM builds
- SSE client uses `URLSession.shared.bytes(for:)` + `bytes.lines` for async line-by-line streaming; set `timeoutInterval = .infinity` for long-lived connections
- `appendDeviceFields(to:boundary:)` is a shared helper for adding sender/target device IDs to multipart POST bodies; chunked uploads add device IDs to the JSON init body instead
- SSE reconnect with exponential backoff: 1s → 2s → 4s → capped at 30s; `startSSE()` is idempotent, call `stopSSE()` before `startSSE()` when credentials change
- Target device selection stored in `ConfigStore.targetDeviceID` (*String? nil = queue/any device*); device list fetched from `GET /api/v1/devices` excluding self
